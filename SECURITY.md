# Security reporting

Do not disclose security vulnerabilities, credentials, tenant identifiers, access tokens, cookies, license tokens, personal data, or confidential URLs in a public GitHub issue.

If GitHub Private Vulnerability Reporting is available for this repository, use **Security** > **Report a vulnerability**. Otherwise, open a minimal public issue stating that you need a private security-reporting channel, without including vulnerability details.

For ordinary product defects without sensitive information, use the repository issue forms.

Only the latest test release is evaluated for security fixes unless a maintainer states otherwise.
