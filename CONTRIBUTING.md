# Contributing

This repository accepts test reports, documentation feedback, and feature requests. It does not accept source-code contributions because the application source is maintained privately.

## Good bug reports include

- the exact release version;
- browser and operating system;
- manager or ordinary-user context;
- reproducible steps;
- expected and actual results;
- sanitized logs or screenshots.

Search for an existing issue before creating a new one. Keep one independently actionable problem per issue.

Do not include confidential or security-sensitive information. Follow [SECURITY.md](SECURITY.md) for vulnerability reporting.
